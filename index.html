How to Setup Your Own Domain Name on Github

Jun 11th, 2014

Github provides such a great hosting service free of charge, too bad that the domain name is default to be

http://github_id.github.io

Indeed, we programmers tend to “show off” our knowledges in anthing-related-to-computer, so it’s often the case that we use our own domain name instead. For easy memorization and, of course, the bragging right.

Here in this post, I will walk through the topic on how to set up your own domain name on Github. Main reason I want to do this is because I found the turtial online is kinda confusing. I will try to explain it in a plain english manner.

If you don’t quite understand the concept, here is a good link that explains everything concept you need to know before get it stared.

Step 1, get a domain name

Yes, you have to buy one and all the domain names come with price. For example, this website’s domain name is 10 dollors/year, not that bad if you think about it. You can buy domain name from domain name registrar, GoDaddy, namecheap just to name a few. For consistency, let’s say you register a domain name yourdomain.com on Namecheap. (Note, this tutorial is for APEX domain name. Subdomains should be able to follow most of the steps until but not include step 4)

Step 2, host your website on Github

For details on how to host your website on github, here is the link. Create repository and push it. Simple enough. By now, you should have a repository called github_id.github.io

Step 3, push a file named CNAME into your respository

Create a file in your github_id.github.io repository called CNAME. There should be only one line in this newly create file, that is youdomain.com. Push this file to Github and make sure it exists under root directory.

Step 4, redirect your domain name to Github

Go to your account where you registered your domain name, select your domain name and under the Host Management tab (or similar tab), click All Host Records. By default, top should have two records, one as @ and one as www. Change them to :

@       192.30.252.154    A(Address)    n/a    1800
www     yourdomain.com    URL Redirect  n/a    1800

In the sub-domain settings, we need to add another record(thanks to github) that redirect the host to Github server, it looks like

@       192.30.252.153    A(Address)    n/a    1800

Save this settings.

Step 5, wait until it comes up

Github can take time to configure these settings, so be patience. To test the host, we can use the methods described in the tutotial.

dig yourdomain.com +nostats +nocomments +nocmd

if the IP addresses show up are precisely 192.30.252.154 and 192.30.252.153(if everything is normal, both should show up), we are good to go!

This is the tutorial for setting up APEX domain name on github, buf if you have a subdomain name, for example www.yourdomain.com instead of yourdomain.com, your don’t need to create A record. Just redirect the www url to your_id.github.io should do the trick.

Here is the original link to the turotial Github provides.

On WWDC14

Jun 4th, 2014

Finished the recap for WWDC 2014 yesterday. Didn’t get a chance to write some of my thoughts on it until today.

Every summer, Google-Apple-Microsoft, the big 3 will host a serise of conferences to “show off” their latest work. Microsoft don’t really draw a lot attention and I personally don’t like its product (despite the fact that I do have a Nokia phone, but it does not run very smoothly and I don’t appreciate that). So, it’s quite interesting to watch Apple WWDC vs. Google I/O each year.

Despite its length (more than 2 hrs), I can see 3 major points from WWDC.

Mac OS X Yosemite
iOS 8
Swift (not Taylor)

The following are my thoughts on each:

Start with Yosemite. I’m not quite “amazed” by this except probably its name. (Dan, never been to Yosemite before but will do in this year.) Updates include a lot “easier to use” features. I will love the features like Mark for email and the new notification ctr where you can put widgets. But still, these new updates do not quite catch my eye when I was watching the video. It’s Apple with Tim but only these updates this year? Not stress on performance/batter-life/compability? Well, let’s wait and see then.

iOS 8 don’t have a lot things as well. Honestly, group-chatting/file-sharing/voice-messge have been out there for years (Yes, I’m talking about WeChat/QQ, which dominate China). I guess next step for iOS is to have it’s own version of moment(a social sharing function built-in WeChat) maybe? One good thing is the tunnel between iOS and Mac OS X is finally opened. I’ve been expecting this for quite a while. The continuity is not a quite new stuff. Last time I heard about this concept is from Netflix, where you can pick-up where you left on video across different platforms. And, it should work well with the large amount of Mac/iOS devices out there. What’s next? Continuity on apps? Wait, this actually could work. You are welcome Apple.

Finally, Swift/DevCenter. As a developer, I gotta say Apple you did a great job here. The ability to use different services from different apps could make our lifes so much easier (both developer and consumer). And the new kits for Graphic looks so cool. 3D kit for casual game? Thats a good idea. Not to mention the new Swift language. Even tho it might be a little bit offend (I know what I’m coding, thank you, I don’t need to see it, just go away Swift!), but it definitely will help in most of the circumanstances. Wow, this new SDK is quite amazing! Can’t wait to start writing my apps, but ya, need to read that 500 pages Swift book first…=P

And, last but not least, no words for iPhone 6. Gosh, I’m waiting for a year but not a single word. (Don’t look at me like this, Nokia. Maybe we should just be friends)

Let’s see what Google I/O will bring =)
